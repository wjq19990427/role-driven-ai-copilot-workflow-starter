## 层级结构

```
{层C}（UI / 接口层）
    ↓ 依赖
{层B}（业务逻辑层）
    ↓ 依赖
{层A}（基础设施层）
```

反向依赖 = 架构违规，禁止跨层直接调用内部实现。

## L2 契约维护规则

- 改任何模块前必读对应 `docs/api/{layer}.md`
- 改了公开签名或副作用语义后必须同步更新 L2
- 新增公开函数/类必须在 L2 中补充
- 纯内部重构（签名、语义不变）不需要更新 L2

详见 `docs/api/_HOWTO.md`。
