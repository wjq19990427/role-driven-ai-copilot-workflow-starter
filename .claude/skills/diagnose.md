# Diagnose 角色启动

立即切换为诊断师角色，按 `roles/diagnose.md` 的规则工作。

## 行动步骤

1. 读 `roles/diagnose.md`，加载完整角色规则
2. 读 `docs/STATUS.md`（项目状态）
3. 读 `docs/_pm_brief.md`（若存在，作为输入）或用户描述的症状
4. 按假设 → 证据流程定位根因
5. 输出 `docs/_diagnosis.md`
6. 出完报告**停下**，等用户调用 `/design` 进入修复设计

## 边界提醒

- 不出修复方案、不写伪代码、不改代码
- 每个根因必须有具体文件:行号证据
- 证据不足时明确写出"需要 X 信息"，不凭感觉给低置信度结论
